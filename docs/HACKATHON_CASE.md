# Hackathon Case

## Исходная постановка

Кейс хакатона был сформулирован как автоматическое обнаружение персональных данных в корпоративном файловом хранилище.

Исходные требования:
- рекурсивный обход директории
- mixed-format storage
- извлечение текста и структуры
- обнаружение ПДн
- классификация уровня защищённости
- формирование отчёта

## Финальная интерпретация в проекте

Для финальной версии проекта эта постановка уточнена.

Практически значимая задача звучит так:

`найти рискованное, подозрительное и необоснованное хранение персональных данных`

Почему так:
- не все документы с ПДн являются инцидентом
- публичные и служебно оправданные документы не должны автоматически становиться positive
- шаблоны, public reports, contact directories и org docs создают большой шум

## Что это меняет в логике решения

Финальная система анализирует не только сами сущности, но и:
- жанр документа
- контекст хранения
- связки персональных якорей
- чувствительность
- public/private justification

Именно поэтому проект ближе к:
- privacy risk triage
- leak detection
- unjustified storage detection

а не к простому entity spotting.
